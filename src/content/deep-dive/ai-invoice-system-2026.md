### Step 2: 適格請求書発行事業者の登録番号検証

受け取ったインボイスの登録番号（T+13桁）が有効かどうかを国税庁のAPIで自動検証します。

```python
import requests

def verify_invoice_number(reg_number: str) -> dict:
    """国税庁APIで登録番号を検証"""
    if not reg_number.startswith("T") or len(reg_number) != 14:
        return {"valid": False, "error": "形式不正"}

    resp = requests.get("https://web-api.invoice-kohyo.nta.go.jp/1/num",
        params={"id": reg_number[1:], "type": "21", "history": "0"})
    data = resp.json()

    if data.get("count", 0) > 0:
        return {"valid": True, "name": data["announcement"][0].get("name", "")}
    return {"valid": False, "error": "登録番号が見つかりません"}
```

freeeやMFでは、この検証が請求書取り込み時に自動実行されます。無効な登録番号が検出された場合は、アラートが表示され、仕入税額控除の対象外として処理されます。

### Step 3: 税率区分の自動チェック

インボイスには8%（軽減税率）と10%（標準税率）の区分が正しく記載されている必要があります。AIは以下のロジックで自動チェックします。

**チェック項目:**

1. **品目名と税率の整合性**: 飲食料品なのに10%が適用されていないか
2. **税額の逆算検証**: 記載された税額が、税抜金額×税率と一致するか
3. **端数処理の検証**: 1インボイスにつき税率区分ごとに1回の端数処理か

```python
def validate_tax_calculation(items: list, stated_tax_8: int, stated_tax_10: int) -> dict:
    """インボイスの税額計算を検証"""

    calc_base_8 = sum(item["amount"] for item in items if item["tax_rate"] == 8)
    calc_base_10 = sum(item["amount"] for item in items if item["tax_rate"] == 10)

    # 税額の逆算（切捨て）
    expected_tax_8 = int(calc_base_8 * 8 / 100)
    expected_tax_10 = int(calc_base_10 * 10 / 100)

    errors = []
    if abs(stated_tax_8 - expected_tax_8) > 1:  # 端数処理で1円の差は許容
        errors.append(f"8%税額不一致: 記載{stated_tax_8}円 vs 計算{expected_tax_8}円")
    if abs(stated_tax_10 - expected_tax_10) > 1:
        errors.append(f"10%税額不一致: 記載{stated_tax_10}円 vs 計算{expected_tax_10}円")

    return {
        "valid": len(errors) == 0,
        "errors": errors,
        "calc_base_8": calc_base_8,
        "calc_base_10": calc_base_10
    }
```
