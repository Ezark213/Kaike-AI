## 要点まとめ（30秒で読める）

- EC事業者の経理は**プラットフォーム手数料・送料・返品処理**が複雑 → AIで仕訳パターンを自動化
- **越境ECの消費税**（輸出免税・リバースチャージ方式）をChatGPTで正確に判定
- Shopify/Amazon/楽天の**売上データをクラウド会計ソフトに自動連携** → 手入力ゼロ
- **在庫評価（先入先出法・移動平均法）の自動計算**でEC事業者の棚卸し作業を80%削減

---

## 背景：EC事業者の経理が複雑な理由

EC事業は取引量が多く、プラットフォームごとに手数料体系が異なるため、経理業務が非常に複雑になります。

**EC経理の特有の課題**:

| 課題 | 詳細 |
|------|------|
| 大量の少額取引 | 1日数十〜数百件の注文 |
| プラットフォーム手数料 | Amazon/楽天/Shopifyで計算方法が異なる |
| 返品・返金処理 | 売上の取消し＋送料負担の仕訳 |
| ポイント・クーポン | 値引きの会計処理 |
| 越境EC | 多通貨・輸出免税・関税の処理 |
| 在庫管理 | SKU数が多く棚卸しが困難 |
| 入金サイクル | プラットフォームごとに異なる入金日 |

## AI EC経理の実践手法

### 1. プラットフォーム別の仕訳自動生成

```
以下のAmazonセラーセントラルの決済レポートデータから、仕訳を自動生成してください。

【Amazonの入金明細（1回の入金サイクル分）】
| 項目 | 金額 |
|------|------|
| 商品売上 | ¥500,000 |
| 配送料（購入者負担） | ¥30,000 |
| ギフトラッピング | ¥5,000 |
| Amazon販売手数料 | -¥50,000 |
| FBA手数料 | -¥35,000 |
| FBA在庫保管手数料 | -¥8,000 |
| 広告費（スポンサープロダクト） | -¥20,000 |
| 返金（3件） | -¥15,000 |
| プロモーション割引 | -¥10,000 |
| **入金額** | **¥397,000** |

【生成してほしい仕訳】
1. 売上計上の仕訳（総額 or 純額、根拠を説明）
2. 各手数料の仕訳
3. 返金の仕訳
4. 入金時の仕訳
5. 消費税の取り扱い（手数料の課税/非課税区分）
```

### 2. 越境ECの消費税判定

```
以下の越境EC取引について、消費税の取り扱いを判定してください。

【取引パターン】
1. 日本のShopifyストアから米国の消費者に直接発送（EMS）
   - 商品: アパレル ¥10,000
   - 送料: ¥2,000

2. 日本のAmazon.co.jpで海外発送対応（FBA海外配送）
   - 商品: 電子機器 ¥30,000

3. 海外のサプライヤーからデジタルコンテンツを仕入れ
   - 月額サブスクリプション: $100
   - サプライヤー所在地: 米国

4. 海外のインフルエンサーへの広告費支払い
   - 金額: $500/月
   - サービス提供地: 米国

5. 日本のAmazon.co.jpマーケットプレイスで海外セラーから仕入れ
   - 商品: ¥50,000
   - セラー所在地: 中国

【各取引について判定してほしい内容】
1. 消費税の課税/免税/不課税/非課税の区分
2. 輸出免税の適用要件（証拠書類の保存義務）
3. リバースチャージ方式の適用有無
4. 仕入税額控除の可否
5. 確定申告書への記載方法
```

### 3. 多通貨の為替換算

```
以下の越境EC取引の為替換算処理を行ってください。

【取引データ（2026年3月）】
1. 3/5: Shopify USD売上 $2,000（注文時レート: ¥148.50）
2. 3/10: PayPal入金 $1,800（入金時レート: ¥149.20）
   ※PayPal手数料$200差し引き
3. 3/15: 中国サプライヤーへの仕入 CNY 50,000（支払時レート: ¥20.50）
4. 3/20: EUR建てのShopifyアプリ利用料 €50（引落時レート: ¥162.00）
5. 3/31: 期末時点の外貨残高
   - USD: $3,000（期末レート: ¥150.00）
   - CNY: ¥0（全額支払済み）

【処理してほしい項目】
1. 各取引の仕訳（円換算額の計算）
2. 為替差損益の計算
3. 期末の外貨建資産の評価替え
4. TTSレートとTTBレートの使い分け
```

```
以下のEC在庫データから、期末の在庫評価を行ってください。

【商品: スマホケースA（SKU: CASE-A001）】

【仕入データ】
| 日付 | 数量 | 単価 | 金額 |

| 8/20 仕入 | 300個 | ¥900 | ¥270,000 |
| 11/10 仕入 | 200個 | ¥880 | ¥176,000 |

【販売データ】
| 月 | 販売数量 |
|----|---------|
| 4-6月 | 150個 |
| 7-9月 | 250個 |
| 10-12月 | 200個 |
| 1-3月 | 150個 |

【期末在庫: 50個】

【評価してほしい項目】
1. 先入先出法での期末在庫金額
2. 移動平均法での期末在庫金額
3. 両方法の売上原価
4. どちらの方法がEC事業者に適しているか（根拠付き）
5. 低価法の適用要否（正味売却価額: ¥750/個の場合）
```

### 5. Shopify×freee連携の設定

EC事業者に最も効果的なのは、プラットフォームとクラウド会計ソフトの自動連携です。

**Shopify → freee連携の手順**:
1. freeeのアプリストアから「Shopify連携アプリ」をインストール
2. Shopifyストアと認証連携
3. 勘定科目のマッピング設定（売上/手数料/送料等）
4. 同期頻度の設定（日次推奨）
5. テスト同期で仕訳の正確性を確認

**Amazon → freee連携**:
1. Amazonの決済レポートをCSVダウンロード
2. freeeのCSVインポート機能で取込
3. 仕訳ルールを設定（自動仕訳パターン登録）
