## 今週のAI会計ニュースTOP5

### 1. 経理白書2026：AI導入済み38%、3年以内に57.8%が導入計画 ── 経理DXは「後戻りなし」

SAP Concurの「経理白書2026」が衝撃のデータを公開した。経理分野のAI「導入済み」は38%、3年以内の導入計画を含めると57.8%に達している。さらに導入企業の54%が週1回以上AIを利用しており、AIは完全に経理業務の「日常ツール」となった。一方で未導入企業では仕訳入力の7割が手作業で、月末残業は平均32時間。格差は拡大の一途だ。

**実務への影響**: 「AI導入38%」は、イノベーション普及の「アーリーマジョリティ」段階に入ったことを意味する。次の2年で過半数到達が見込まれ、「AIを使わない経理」は競争劣位に。経理部門のAI投資は「検討事項」から「経営課題」に格上げが必要だ。

### 2. Basis AI、$100M調達で$1.15Bユニコーン ── Form 1065の完全自律処理を世界初実証

会計AI特化のスタートアップBasisが$100M（約150億円）を調達し、$1.15B（約1,725億円）のユニコーン企業となった。Accel・Google Ventures・Khosla Venturesが出資。既にTop 25会計ファームの30%が顧客で、世界初のForm 1065（パートナーシップ確定申告）の完全自律AIエージェント処理を実証した。同時期にAccrualも$75Mを調達し、確定申告の準備時間85%削減を実績として示した。

**実務への影響**: 会計AIスタートアップへの投資が2月だけで$175M（約260億円）に達し、2026年は「会計AIエージェント元年」と呼べる状況。日本市場でも法人税・消費税申告の定型部分のAI自動化が加速する見通し。

### 3. 弥生「AI搭載うたわず」のDX戦略 ── freee・MFと真逆のアプローチ

弥生株式会社が、freeeやMFがAI機能を前面に打ち出す中、「AI搭載をうたわない」戦略を展開。弥生会計NextにはAI取引入力β版やAI資金予測β版を実装しながら、マーケティングでは「かんたん・やさしい」を訴求。ITリテラシーに不安のある中小企業に寄り添う設計思想を貫いている。

**実務への影響**: 「AIを使いこなす」スキルが不要な設計は、人材不足の中小企業に最適。一方で、AIの判断をブラックボックスとして受け入れるリスクもあり、税務上重要な判断（消費税区分・減価償却方法）の検証プロセス整備が重要。

### 4. Anthropic、Claude Cowork Financeで経理ワークフロー刷新 ── PwCと連携でエンタープライズ展開

AnthropicがClaude Coworkの経理・財務向けプラグインを発表。Claude Opus 4.6をエンジンに、請求書と元帳の突合、月次レポート作成、監査対応資料の準備をAIエージェントが横断処理する。ExcelからPowerPointへのクロスアプリケーション処理も実現。PwCとのパートナーシップでエンタープライズ展開を加速。

**実務への影響**: 「ツール間の手作業」がAIエージェントに置き換わる時代の到来。ただし財務数値の自動転記は誤りが致命的なため、「ヒューマン・イン・ザ・ループ」設計が不可欠。

### 5. PwC J-SOX自動化 + AI Factory設立 ── Big4の税務AI戦略が本番運用へ

PwC Japanが今週2つの動きを見せた。①J-SOX一次評価の生成AI自動化で数百〜数千時間の削減を実証。②税務AI専任組織「AI Factory」を発足し、Tax Guidance Assistant・移転価格文書自動化・J-SOX効率化を推進。Big4全社がAI活用を本格化し、監査×AIは「標準装備」の時代に突入。

**実務への影響**: Big4のAI組織化は、税務AI活用が「実験フェーズ」から「本番運用」に移行した証左。中堅税理士事務所もAI導入の具体的計画が急務に。

## 【実践ガイド】経理AI導入ロードマップ ── 「何から始めるべきか」を3ステップで解説

経理白書2026で「38%が導入済み」とわかった今、まだ導入していない62%の企業は焦りを感じているかもしれません。しかし、焦って導入すると失敗します。以下の3ステップで段階的に進めましょう。

### ステップ1: 現状の業務フローを棚卸しする

まず、自社の経理業務で「最も時間がかかっている作業」を特定します。

**使用プロンプト:**
```
あなたは経理業務の効率化コンサルタントです。
以下の経理部門の業務フローを分析し、AI自動化の優先順位をつけてください。

【入力情報】
- 会社規模: [従業員数 / 年商]
- 経理担当者数: [人数]
- 使用会計ソフト: [freee / MF / 弥生 / その他]
- 月末締め作業に要する日数: [日数]
- 最も時間がかかる業務: [自由記述]

【出力形式】
1. AI自動化の優先度（高・中・低）と理由
2. 各業務の推定削減時間
3. 推奨ツール（3段階: 無料→低価格→本格導入）
4. 3ヶ月後の目標設定
```

### ステップ2: 小さく始めて効果を測定する

いきなり全業務をAI化するのではなく、1つの業務に絞って効果を測定します。

**推奨の「最初の一歩」:**

| 企業規模 | 推奨する最初のAI導入 | ツール | 月額 |
|---------|---------------------|-------|------|
| 個人事業主 | レシート撮影→自動仕訳 | freee / MF（スマホアプリ） | 無料〜¥2,680 |
| 10人以下 | 請求書受領→自動処理 | TOKIUM / invox | ¥10,000〜 |
| 50人以上 | 経費精算→AI審査 | SAP Concur / TOKIUM | ¥30,000〜 |

**効果測定プロンプト:**
```
以下のAIツール導入前後のデータを比較し、効果を定量評価してください。

【導入前】
- 対象業務: [業務名]
- 月間処理件数: [件数]
- 1件あたり処理時間: [分]
- 月間エラー率: [%]

【導入後（1ヶ月目）】
- 月間処理件数: [件数]
- 1件あたり処理時間: [分]
- 月間エラー率: [%]
- ツール月額コスト: [円]

出力: ROI計算（時間削減額 vs ツールコスト）、改善率、半年後の見通し
```

### ステップ3: 「AI×人間」のルールを決める

AIは便利ですが、全てを任せるのは危険です。特に税務に関わる判断は、人間のレビューが必須です。

**チェックポイント設定プロンプト:**
```
経理業務においてAIが自動処理した結果を人間がレビューすべきポイントを設計してください。

【条件】
- 会計ソフト: [ソフト名]
- AI自動化している業務: [業務名]
- 税務上のリスクが高い取引: [例: 交際費、海外取引、固定資産]

【出力形式】
1. 必ず人間がチェックすべき項目（消費税区分、勘定科目の例外的判断等）
2. AIに任せてよい項目（過去パターンと一致する定型取引等）
3. 月次・四半期・年次で実施すべきAI出力の検証手順
4. 「AIが間違いやすい」パターン一覧
```
